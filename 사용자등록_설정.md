# 🔧 사용자 등록 기능 — Firestore 규칙 추가

새로 추가된 **사용자 등록 시스템** 을 사용하려면, Firestore 보안 규칙에 `/users` 컬렉션을 추가해야 합니다. **딱 한 번만** 작업하면 됩니다 (약 2분).

---

## 1단계. Firestore 규칙 업데이트

1. https://console.firebase.google.com → `arisu-central` 프로젝트
2. 좌측 **"Firestore Database"** → 상단 **"규칙"** 탭
3. 편집기의 **기존 코드 모두 삭제**
4. 아래 규칙을 **그대로** 붙여넣기:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // 설정 (비밀번호, URL, 지시번호 접두어)
    match /config/{doc} {
      allow read, write: if request.auth != null;
    }

    // 관리자 PC 등록
    match /adminDevices/{deviceId} {
      allow read, write: if request.auth != null;
    }

    // 사용자 등록 (관리자가 미리 등록한 사용자만 로그인 가능)
    match /users/{userId} {
      allow read, write: if request.auth != null;
    }

    // 공사지시
    match /orders/{orderId} {
      allow read, write: if request.auth != null;

      match /chat/{messageId} {
        allow read, write: if request.auth != null;
      }
    }
  }
}
```

5. **"게시"** 클릭
6. 약 1분 기다리기

---

## 2단계. 직원 명단 등록

1. 본인 PC에서 **관리자로 로그인**
2. 우측 상단 **⚙️ 관리자** 클릭
3. **👥 등록된 사용자** 섹션 찾기 (공유 주소 바로 아래)
4. 직원 이름 입력 → **➕ 추가** (또는 엔터)
5. 5~8명 직원 모두 추가

> 💡 **중요**: 직원이 평소 입력하는 **이름과 정확하게 같게** 등록해야 합니다.
> 예: 직원이 "김철수" 로 입력하는데 관리자가 "김 철수" (공백) 로 등록하면 로그인 안 됩니다.

---

## 3단계. 기존 사용자 안내

규칙 적용 후, **이미 사용 중이던 직원들은 등록되어 있지 않아서 로그인이 안 됩니다.**

관리자가 직원 명단을 모두 등록할 때까지는 직원들이 로그인 시 다음 메시지를 보게 됩니다:

> 🚫 등록되지 않은 사용자입니다. 관리자에게 등록을 요청하세요.

해결: 관리자가 직원 이름을 빠르게 모두 등록 → 직원들에게 "재접속하세요" 라고 안내.

## 한계 & 권한 한도

| 항목 | 한도 |
|---|---|
| 일반 사용자 | 최대 **8명** |
| 관리자 PC | 최대 **3대** |
| 관리자 (사용자 등록과 무관) | 관리자 비밀번호로 별도 접속 |

> 💡 **관리자는 등록 안 해도 됩니다.** 관리자 비밀번호로 들어오면 사용자 목록과 무관하게 접속 가능합니다.

## 자주 묻는 질문

**Q. 관리자 본인은 사용자 목록에도 등록해야 하나요?**
A. 안 해도 됩니다. 관리자 비밀번호로 들어오면 자동으로 모든 기능 사용 가능합니다. 다만 관리자 비밀번호를 잊거나 평소엔 일반 사용자처럼 쓰고 싶다면 본인도 등록해 두는 게 좋습니다.

**Q. 8명 한도가 너무 적은데요**
A. HTML 파일의 `if(snap.size >= 8)` 부분에서 숫자를 늘리시면 됩니다. (코드 두 곳: 8 → 원하는 숫자) Firebase 무료 한도는 충분히 더 받아도 됩니다.

**Q. 한 명을 두 번 등록할 수 있나요?**
A. 안 됩니다. 같은 이름은 중복 방지됩니다.

**Q. 사용자를 제거하면 그 사람이 등록한 공사지시도 삭제되나요?**
A. 아니요. 공사지시는 그대로 남고, 그 사람이 다시 로그인을 못할 뿐입니다.

**Q. 관리자 PC 3대도 부족하면?**
A. HTML의 `if(all.size >= 3)` 숫자를 늘리시면 됩니다. 다만 관리자는 보안상 3대 정도가 적당합니다.
